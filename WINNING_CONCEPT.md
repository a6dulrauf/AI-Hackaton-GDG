# Refined Concept — How We Win

## The one-liner (use as the project title / opening line)
**LifeLine — the missing interface to Al-Khidmat's donor network, built to *protect* it.**
*"The bottleneck isn't the data — it's the interface between a stressed family and 25,000 verified donors. We built that interface."*

## The positioning shift (free, biggest single lever)
Most teams will build "a search over a donor list." We frame it as solving the
**partner's own stated problem**: the interface, and doing it **without spamming or
burning out the network**. Quote Al-Khidmat back to the judges in the first 15 seconds.
This reframes a CRUD app into the exact thing the partner asked for — and adds an
ethical, human dimension judges remember.

## What changed vs. the first plan
Three features move **from "stretch" into the committed core**, because they are the
differentiators that separate a winner from a working demo. They are cheap and visible.

### Core now includes (in addition to C1–C5):
- **C6 — "Why this donor" rationale.** Every recommended donor shows a plain-English
  reason ("closest eligible O− donor, last gave blood 6 months ago, responds 80% of the
  time"), not just a score. Already generated by `ranking.py` (`why` field). ~Free.
- **C7 — Donor-fatigue / network-health indicator.** The UI visibly *skips* and labels
  recently-contacted donors, and shows a small "network health" meter. This is the single
  clearest proof we understood "without burning out frequent responders." ~30 min.

### Still stretch (only if core is green):
- **S — Urdu voice intake** (Groq Whisper). Genuine wow, ~40 min.
- **S — Blood-group compatibility fallback** (already supported by `ranking.py`
  `allow_compatible=True`). ~10 min to surface in UI.
- **S — Map dispatch animation.**

## The four things judges actually score (design every choice around these)
1. **It works live** — hand a judge the keyboard; let them type a messy Urdu/English request.
2. **It tells a story** — the LifeLine framing + the human-network angle.
3. **One number that lands** — the **Spam Shield**: "A blast would have messaged 240
   people. We contacted 9." Say it out loud.
4. **It solves the partner's pain** — the interface, and protecting the network.

## Updated feature priority (build top-down; stop where time runs out)
1. C1 Natural-language intake (Groq, mixed language)
2. C2 Eligibility + composite ranking
3. C3 Chat UI + ranked donor list
4. C4 Leaflet map (hospital + donor pins)
5. C5 Wave outreach + live "X of Y confirmed" + final summary
6. C6 "Why this donor" rationale (surface the `why` field)
7. C7 Donor-fatigue indicator + Spam Shield stat

9. S Compatibility fallback toggle
10. S Map dispatch animation

## Deliberately NOT building (say "roadmap", never build)
Real WhatsApp/SMS, database, auth, multi-city scale, pixel-level fraud detection.

## 30-second pitch script
> "When someone needs blood in Karachi tonight, the data already exists — Al-Khidmat has
> 25,000 verified donors. The problem is the *interface*: reaching the right few without
> spamming thousands or burning out the people who always say yes. LifeLine is that
> interface. You type — in Urdu, English, however you're panicking — and it finds the
> closest eligible donors, explains *why* each one, contacts them in small waves, and
> escalates only if needed. A blast would have messaged 240 people. We messaged 9, and
> got 5 to commit. Want to try it? Type your own request."
